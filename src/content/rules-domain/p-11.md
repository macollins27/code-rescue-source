Every data access path MUST call canAccess() from @repo/api/permissions before returning data. The default branch is DENY — absence of an allow rule means denial, not allowance. Per system-map.md: "auditors check authorization completeness." Reviewer Pass 7 enforces by negation — a procedure that does not call canAccess() is presumed to leak.
